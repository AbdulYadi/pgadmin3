Autentizace metodou Ident selhala

Server není ochoten přijmout aktuálního uživatele: Server hlásí

Pokud se objeví toto hlášení, je v pg_hba.conf záznam pro kombinaci klienta / uživatele / databázi s nastaveným ověřování na „ident“. Některé distribuce, např. Debian, mají tuto možnost jako výchozí. Aby mohlo ověření typu ident proběhnout v pořádku, jsou zapotřebí doplňující nastavení. Podrobnosti najdete v nápovědě k PostgreSQL. Pro začátečníky může být vhodnější použít jiný typ ověřování. Dobrou volbou jsou hesla šifrovaná pomocí MD5. V pg_hba.conf je můžete nastavit nějak takto:

host all all 192.168.0.0/24 md5

Tento příklad uděluje na základě hesla šifrovaného pomocí MD5 přístup ke všem databázím pro všechny uživatele v privátní síti 192.168.0.0/24.

K úpravě souboru pg_hba.conf můžete použít editor nastavení, který je součástí pgAdmin III. Po provedení změn v pb_hba.conf je potřeba přimět server, aby nové nastavení načetl. Buď pomocí pg_ctl nebo restartováním procesu serveru.
