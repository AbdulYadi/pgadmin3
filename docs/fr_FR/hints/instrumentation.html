Outils serveur

adminpack est installé et activé par défaut si vous avez utilisé l'installeur one-click de PostgreSQL pour Windows et est inclus en tant que module contrib pour toutes les versions de PostgreSQL, à partir de la version 8.2. Néanmoins si vous avez une version antérieure, vous devrez l'installer manuellement. Pour cela, copiez le code source approprié dans le répertoire contrib de votre répertoire des sources de PostgreSQL. Vous pouvez télécharger les sources ici.

Exécutez les commandes suivantes (en substituant admin par admin81 si nécessaire) :

    # cd $PGSRC/contrib/admin
    # make all
    # make install

Le module doit maintenant être construit et installé dans votre installation de PostgreSQL. Pour être utilisé, vous devez installer le module dans la base de données que vous utilisez come base de maintenance (habituellement postgres, bien que vous pouvez avoir sélectionné une autre base de données). Pour cela, trouvez le fichier admin.sql ou admin81.sql installé lors de l'étape précédente - il devrait normalement se trouver dans le répertoire /usr/local/pgsql/share/contrib/ et exécutez-le sur votre base de données. Vous aurez normalement besoin de le faire en tant que superutilisateur PostgreSQL :

    $ psql postgres < /usr/local/pgsql/share/contrib/admin.sql

La prochaine fois que vous vous connecterez à pgAdmin, il détectera la présence de l'adminpack et disposera de toutes les fonctionnalités.

Les outils serveur étendus ne sont pas supportés pour les versions 7.3 et 7.4 de PostgreSQL.
