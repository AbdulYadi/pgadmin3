Modifier les données d'une table sans clé primaire

Comme la table ne dispose pas de clé primaire ou d'OID, vous pouvez seulement visualiser les données. Insérer de nouvelles lignes et modifier les lignes existantes n'est pas possible pour l'outil Modification des données sans clé primaire.

Pour modifier les données, pgAdmin III a besoin d'une clé primaire sur la table, ce qui est une bonne pratique lors de la conception d'une base de données. Sinon, la table peut être créée avec l'option WITH OIDS. Notez que les OID n'ont aucune garanties d'être uniques sur une très longue période de temps, donc utiliser des OID comme clé primaire est seulement un choix secondaire.
