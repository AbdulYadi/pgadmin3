Autenticación de Ident falló

El servidor no acepta el usuario actual. El servidor reporta

Si este mensaje aparece, la línea del archivo pg_hba.conf encontrada para la combinación de su cliente / usuario / base de datos está configurada para autenticación "ident". Algunas distribuciones, p.e. Debian, tiene esto por defecto. Para realizar autenticación basada en ident exitosamente, necesita configuración adicional; vea la ayuda de PostgreSQL para ello. Para un principiante, peude ser más apropiado usar un método de autenticación diferente; contraseñas encriptadas con MD5 son una buena opción, las cuales pueden ser configurdas mediante una entrada en pg_hba.conf como esta:

host all all 192.168.0.0/24 md5

Este ejemplo otorga acceso a través de contraseñas encriptadas MD5 a todas las bases de datos y todos los usuarios en la red privada 192.168.0.0/24.

Usted puede utilizar el editor de pg_hba.conf que está incluido en pgAdmin III para editar el archivo de configuración pg_hba.conf. Luego de modificar pg_hba.conf, necesita disparar una recarga de la configuración del servidor utilizando pg_ctl o deteniendo y reiniciando el proceso de servidor.
