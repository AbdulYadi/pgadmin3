Editando Vistas, Procedimientos Almacenados o Funciones

Al editar el código de una vista, procedimiento almacenado o función, en vez de usar el pequeño cuadro de texto en el diálogo de propiedades del objeto, puede encontrar más conveniente usar las opciones de scripting en pgAdmin para editar el objeto en la herramienta de consultas. Esto le da el beneficio de las características del editor de la Herramienta de Consultas tales como autocompletado, buscar y reemplazar y una mayor área de trabajo.

Para usar las opciones de scripting, en la ventana de explorador de pgAdmin, haga click-botón-derecho sobre el objeto que desea editar, y seleccione la opción sentencia CREATE. Puede estar en el submenu Scripts. Alternativamente, puede seleccionar el objeto y luego elegir la opción setencia CREATE del submenú Scripts del menú Herramientas.

Edite la definición del objeto como se requiera, y use el botón Ejecutar Consulta para aplicar sus cambios. No olvide que modificar el nombre de una vista o función, o la firma de una función o procedimiento puede provocar que se cree un nuevo objeto diferente del original. Este útil efecto colateral lo hace conveniente para crear objetos nuevos similares a otros. Puede necesitar refrescar el explorador de pgAdmin para mostrar los nuevos objetos creados.

Usted puede utilizar esta técnica para realizar todo tipo de operaciones en diferentes objetos, sin embargo la sintaxis CREATE OR REPLACE de PostgreSQL la hace particularmente útil para editar vistas, procedimientos almacenados y funciones.
