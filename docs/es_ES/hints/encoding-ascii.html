Codificación de base de datos

La base de datos es creada para almacenar datos usando la codificación SQL_ASCII. Esta codificación está definida para carácteres de 7 bit solamente; el siginificado de carcarteres con el 8vo bit encendido (caracters no-ASCII 127-255) no esta definido. En consecuencia, no es posible para el servidor convertir estos datos a otras codificaciones.

Si usted está almacenando datos no-ASCII en la base de datos, lo instamos enérgicamente a que use una codificacion de base de datos adecuada representando su conjunto local de caracteres para aprovechar los beneficios de la conversión automática a diferentes codificaciones de clientes cuando sea necesario. Si almacena datos no-ASCII en una base de datos SQL_ASCII, puede encontrar caracteres extraños escritos o leídos de la base de datos, causados por problemas de conversion de codificación. Esto puede causarle muchos dolores de cabeza al acceder a la base de datos usando diferentes programas clientes y manejadores.

Para la mayoría de las instalaciones, la codificación Unicode (UTF8) proveerá la mayor flexilibilidad.
