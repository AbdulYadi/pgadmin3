Views, Stored Procedures oder Functions bearbeiten

Anstatt zum Bearbeiten von Views, Stored Procedures oder Functions das kleine Text-Feld des Eigenschaften-Dialog des des Objektes zu verwenden, werden Sie deutlich angenehmer finden die Skripting-Funktionalitäten des Query-Editors zu verwenden. Sie können hier von einem größeren Arbeitsbereich profitieren und die Funktionen Auto-Complete oder Suchen-und-Ersetzen nutzen.

Um die Scripting Funktion zu verwenden machen Sie einen Rechts-Klick auf das zu bearbeitende Objekt im pgAdmin Browser und wählen Sie CREATE script. Diese Funktion könnte sich im Untermenu Scripts befinden. Alternativ können Sie die Funktion CREATE script über den Menupunkt Werkzeuge - Scripts erreichen.

Bearbeiten Sie das Objekt nach Ihren Wünschen und drücken Sie den Execute Query um Ihre Änderungen anzuwenden. Bitte beachten Sie, dass die Änderung der Signatur oder die Änderung des Stored Procedure oder Function Namens dazu führt, dass ein neues Objekt angelegt wird. Dieser nützliche Seiteneffekt macht es seht einfach neue aber ähnliche Objekte anzulegen. Um die neu angelegten Objekte zu sehen kann es notwendig sein die Anzeige den pgAdmin Browsers aufzufrischen.

Sie können diese Technik für alle Arten von Operationen für die jeweiligen Objekte verwenden. Allerdings ist die CREATE OR REPLACE Syntax von PostgreSQL besonders nützlich für das Bearbeiten von Views, Stored Procedures und Functions.
